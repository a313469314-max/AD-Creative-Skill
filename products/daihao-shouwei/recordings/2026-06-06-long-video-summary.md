## 2. 核心玩法判断

《代号守卫》早期录屏呈现的是一套“塔防战斗 + 关后养成 + 地图推进”的循环：

```text
进入关卡
-> 敌人沿固定路径推进
-> 玩家布置/召唤/强化防守单位
-> 守住水晶或防守核心
-> 胜利结算并获得奖励
-> 回到地图/主城
-> 替换装备、升级装备、调整队伍或上阵英雄
-> 继续挑战下一关
```

这条长录屏覆盖的是新手早期流程，重点不是开放式建造或大地图策略，而是竖屏路径塔防、局内选择、通关奖励和关后变强。

## 3. 战斗内循环

战斗内循环可以拆成六个动作节点：

1. **进入关卡**：玩家从地图或加载页进入沙漠主题战斗场景。
2. **识别路线**：敌人从上方或路线入口出现，沿固定道路向下方水晶/核心推进。
3. **布置防线**：玩家根据引导点击底部按钮、卡牌或召唤/强化入口，让防守单位逐渐上场。
4. **应对波次**：敌人密度上升，场上出现火焰、冰晶、投射物、范围攻击、血条和红色提示。
5. **处理局内事件**：战斗中可能出现三选一事件、怪物增强提示或“幸运”奖励弹窗。
6. **完成结算**：守住核心后出现“战斗胜利”，展示血量剩余、MVP/输出统计、金币/经验/道具奖励。

当前录屏能确认战斗包含波次推进、资源/按钮、暂停/倍速、局内选择、胜利结算这些基础系统。单位具体名称、职业、技能树、伤害规则和失败惩罚没有被完整证明。

## 4. 战斗外循环

战斗外循环主要围绕关后成长：

1. **地图推进**：战斗结束后回到沙漠章节地图，地图上有节点、星级/进度和“开始挑战”类按钮。
2. **装备替换**：video-02 和 video-04 都出现装备对比，能确认新装备相较旧装备有攻击数值提升，例子包括攻击 10 -> 27。
3. **装备升级**：video-02 展示了装备从等级 0 到等级 1 的升级反馈，并消耗金币或材料。
4. **队伍/英雄管理**：录屏中出现队伍编辑、角色卡、槽位、战力加成、英雄入口和布阵页面。
5. **英雄获得与上阵**：video-06 出现“获得史诗英雄”以及新英雄上阵引导，完成后提示“保存布阵成功”。
6. **继续挑战**：成长操作后，玩家回到地图或战斗准备面板，再次进入下一关。

当前录屏能确认早期流程中存在装备、队伍、英雄、布阵和地图推进。但英雄稀有度体系、抽卡/招募规则、装备掉落来源、升星/技能长期养成没有完整证明。

## 5. 新手体验节奏

长录屏的新手节奏是先让玩家进入战斗，再逐步打开成长系统：

```text
首关战斗教学
-> 第一关后装备/队伍成长
-> 第二关完整战斗
-> 第二关后主界面、装备、商业化入口露出
-> 第三关更长战斗
-> 第三关后获得史诗英雄并上阵
```

这个节奏说明游戏早期不是长时间停留在菜单教学，而是先用战斗建立玩法，再在关后补充装备、队伍、英雄和商业化入口。录屏中每一轮战斗后都有一个“变强或推进”的反馈点，形成继续下一关的动机。

- 题材类型：轻奇幻/卡通冒险。

- 美术风格：竖屏 2D Q 版卡通，角色和敌人都是小比例单位，整体明亮、轻度、移动端友好。
- 战斗场景：沙漠峡谷路线、城门/据点、底部水晶核心、路径两侧防守点。
- 地图场景：沙漠章节节点地图，有已通关/待挑战节点、主按钮和底部功能栏。
- 角色/单位：人类剑士、蓝色法师/骑士类单位、绿色弓手/植物类单位、火焰/炮塔类单位、亡灵/骷髅类角色、宠物/怪物形象。
- 敌人/Boss 视觉：多种小怪沿路线推进；video-06 出现红龙和城堡场景，但当前证据不足以确认完整 Boss 战规则。

## 8. 可以沉淀的产品事实

- 这是竖屏移动端录屏，早期核心玩法是路径塔防。
- 战斗目标是防守固定路线上的核心/水晶，敌人沿路线分波推进。
- 玩家在战斗中通过按钮、卡牌或引导进行布置、召唤、强化等操作。
- 战斗中存在三选一事件或奖励选择，选择后可能影响后续局势。
- 胜利后有结算页，包含剩余血量、输出/MVP 统计和多种奖励。
- 战斗后会回到地图或主城，并进入装备、队伍、英雄等成长操作。
- 装备存在替换和升级，能看到明确的攻击数值提升。
- 英雄系统存在“获得史诗英雄 -> 引导上阵 -> 保存布阵成功”的早期流程。
- 早期 UI 会露出月卡、免广告卡、首充、限时 UP、福利/活动等入口，但录屏没有展开完整规则。

# 游戏概念分镜提示词方法（对外分享版）

## 适合用在什么上面

适合用于游戏概念试玩视频、假实机分镜图、AI 出图 prompt、图生视频 prompt 和广告分镜预演。尤其适合以下场景：

- 产品还没有足够实机录屏，但需要快速验证广告视觉方向。
- 需要把“游戏卖点”变成一组看起来像真实游戏流程的画面。
- AI 出图总是像概念海报，不像游戏截图。
- 分镜之间缺少任务承接，看起来像随机美术图堆叠。
- 想让外包、AI 或团队成员按统一结构生成分镜。

## 一句话核心

这套方法的价值不是词库，而是把广告分镜做成一组可连续观看的概念试玩流程：每一帧都像真实游戏录屏中的一个阶段，有镜头位置、玩家目标、UI 反馈、玩法状态和前后承接。

## 标准分镜流程

优先按真实游戏体验链路组织，而不是按美术场景随机堆图。

| 阶段 | 画面目标 | 必备信号 |
| --- | --- | --- |
| 登录 / 选角 | 建立题材、品质和主视觉 | 主视觉（Key Visual）、角色阵容、标题 Logo、开始游戏、服务器栏 |
| 城镇 / 找 NPC | 证明这是可操作游戏世界 | 第三人称角色、任务追踪、NPC、地图、移动摇杆 |
| 地图探索 | 展示世界规模和探索欲 | 远景地标、目标距离、开放空间、玩家背影 |
| 对话 / 接任务 | 建立剧情动机 | NPC 名称、对话框、任务目标、角色站位 |
| 小怪战 | 展示即时反馈 | 技能按钮、伤害数字、敌方血条、状态图标 |
| Boss 战 | 制造强压力和高价值挑战 | Boss 血条、队友支援、连击数、强特效 |
| 挑战成功 | 释放兑现反馈 | 胜利标题、奖励、伤害统计、领取按钮 |
| 抽卡 / 招募 | 转向商业化与收集欲 | 卡池、稀有度、资源栏、保底提示、十连按钮 |

## 单张分镜 prompt 结构

生成单张分镜图时，按以下顺序写：

```text
1. 画面用途
   说明这是哪类游戏界面：登录界面、实机探索截图、战斗截图、胜利结算、抽卡界面。

2. 画面规格
   写清比例、宽屏或竖屏、是否手机游戏 UI、是否第三人称视角。

3. 镜头语言
   写清远景 / 中景 / 近景、背视角 / 越肩视角 / 俯视角、玩家和主要目标的位置。

4. 场景设定
   写清地点、空间层次、地标、环境状态和气氛。

5. 主体角色
   写清玩家角色、NPC、敌人或核心展示角色的外观、姿态、动作和关系。

6. 构图关系
   写清前景、中景、远景分别承担什么信息。
   单张图必须有明确主焦点。

7. 玩法 UI
   按当前分镜阶段补齐 HUD、任务、技能、血条、奖励、资源、按钮等。
   UI 只承担当前阶段的系统证明，不能和过多剧情主体抢焦点。

8. 美术风格
   写清题材风格、渲染质量、材质、光影、体积雾、粒子、电影感等。

9. 色彩体系
   指定主色和辅助色，并说明服务的情绪。

10. 质量约束
   强调高保真 3D、真实游戏截图、UI 稳定、焦点清晰、不要扭曲。
```

## 可复用分镜模块

### 登录 / 选角

#### 适合用在什么上面

适合用于第一帧建立产品卖相、题材气质、角色吸引力和广告视觉质量。

#### 必须写清

- 主视觉角色阵容，通常 1-3 个核心角色。
- 上方标题 Logo 预留或直接生成。
- 下方开始游戏、服务器选择、账号协议。
- 两侧系统图标、设置、公告、客服、年龄评级。
- 背景必须是世界观地标，不要只是纯氛围图。

### 探索 / 跑图

#### 适合用在什么上面

适合证明“这是能玩的世界”，而不是单张概念海报。

#### 必须写清

- 第三人称或越肩视角，玩家背对镜头。
- 玩家面向远处目标，形成探索动机。
- 左上小地图、左侧任务追踪、左下虚拟摇杆、右下技能或动作按钮。
- 画面中央可以放目标距离、任务指引或 NPC 标记。

### 对话 / 接任务

#### 适合用在什么上面

适合承接剧情和任务，让玩家行为有原因。

#### 必须写清

- 玩家、NPC、关键目标三者站位。
- NPC 头顶名称或任务标识。
- 对话框、任务标题、接受 / 继续按钮。
- UI 比战斗更轻，重点是任务动机。

### 战斗 / Boss

#### 适合用在什么上面

适合表达爽感、压力、操作反馈和高价值挑战。

#### 必须写清

- 玩家动作、敌人分布、技能爆发位置。
- 敌方血条、红色名称、伤害数字、状态图标。
- Boss 战需要顶部 Boss 名称和长血条。
- 技能按钮要带冷却、普通攻击、闪避、目标切换。
- 特效要服务动作反馈，不要遮住全部 UI 和主体。

### 结算 / 奖励

#### 适合用在什么上面

适合做兑现反馈，证明前面的挑战有结果。

#### 必须写清

- 顶部胜利大字和击败对象。
- 中央奖励栏，展示资源、装备、宝箱、经验。
- 左侧或右侧战斗统计：总伤害、最高连击、战斗时长。
- 领取奖励、继续、倒计时按钮。
- 角色胜利姿态与敌人崩解，形成结果证明。

### 抽卡 / 招募

#### 适合用在什么上面

适合承接收集欲、稀有资产展示和商业化动机。

#### 必须写清

- 中央角色展示和稀有度。
- 左侧卡池列表或 UP 标识。
- 顶部货币资源栏和充值加号。
- 底部单抽 / 十连按钮。
- 保底提示、概率公示、滚动公告、礼包入口。

## 图生视频补充

从分镜图转视频时，不要重写成美术 prompt，要补充运动层。

建议结构：

```text
将这张游戏实机截图变成一段真实游戏录屏。
镜头缓慢向前推进 / 轻微环绕 / 稳定跟随角色。
角色有轻微呼吸、衣摆、发丝或装备摆动。
NPC 或敌人有缓慢走动、待机动作或攻击动作。
环境有云雾流动、光影变化、粒子飘动。
保持游戏 UI 稳定，不要扭曲，不要变形。
保持角色外观一致，画面像真实游戏录屏，流畅高清。
```

## 使用检查表

输出分镜 prompt 前检查：

- 这张图属于游戏流程中的哪个阶段？
- 画面是否能让用户一眼看出“这是游戏截图”？
- UI 是否和当前玩法阶段一致？
- 玩家目标、阻碍或奖励是否可见？
- 前后分镜是否有剧情、任务或玩法承接？
- 特效是否服务反馈，而不是只做视觉堆砌？
- 单帧是否能容纳当前信息量？
- 这张图的主视觉、主动作、主信息点和主反馈是否各只有一个？
- 宏观叙事和精细操作是否被写进了同一张图？
- 如果要转视频，是否单独写了镜头运动、角色运动、环境运动和 UI 稳定要求？

## 和效果广告创意的关系

本方法主要解决“分镜图如何像游戏、如何连续、如何可复用”的问题。

生成广告脚本时，还需要额外判断：

- 开头钩子段是否有足够强的停留理由。
- 是否有故事冲突。
- 核心变化是否有触发机制。
- 素材是否承担用户定位。
- 玩法承接是否足够，不只是最后闪 UI。

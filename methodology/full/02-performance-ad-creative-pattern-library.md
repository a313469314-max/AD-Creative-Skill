# 效果广告创意方法卡片库（对外分享版）

## 适合用在什么上面

适合用于游戏广告脚本发散、创意评审、素材复盘和方法培训。它特别适合帮助团队回答：

- 为什么这个开头能抓停留？
- 为什么这个脚本看起来完整但不想下载？
- 为什么画面很酷但后端质量不稳定？
- 为什么用户看懂了功能，却没有情绪参与？
- 一个产品卖点可以如何拆成多种广告结构？

使用时不要直接问“套哪个模板”，而要先判断素材当前缺什么：缺好奇心、缺冲突、缺反馈、缺视觉刺激、缺产品承接，还是筛错用户。

## 方法 1：同源多结果法

### 核心结构

```text
同一个起点
-> 不同处理方式 / 不同条件
-> 产生完全不同结果
-> 最后做横向对比
```

### 适合用在什么上面

适合开蛋、开箱、抽卡、召唤、合成、进化、装备打造、角色转职、宠物培养等有“同一起点，不同结果”的系统。

### 为什么有效

用户天然想知道“如果换一种方式，会出现什么结果”。当起点相同、结果不同，用户会因为好奇心继续看下一个结果。

### 使用检查问题

- 用户能否在开头看懂这是同一个起点？
- 每次变化后是否立刻给出结果？
- 结果之间是否有明显外观、体型、稀有度或能力差异？
- 最后有没有横向对比，把差异放大？
- 是否只是展示差异，却缺少情绪推进？

### 失效风险

- 一次展示太多结果，用户记不住。
- 差异不明显，看起来像重复画面。
- 只有展示，没有故事或选择动机。
- 广告暗示了游戏中不存在的具体机制。

一句话总结：用同一个起点制造多个结果，让用户因为“还想看下一个结果”而停留。

## 方法 2：被低估资产逆袭法

### 核心结构

```text
低价值开局 / 被迫接受差选择
-> 被否定、被轻视、没资源
-> 坏局里出现隐藏成长线
-> 弱小资产逐步变强
-> 最后完成逆袭
```

### 适合用在什么上面

适合弱变强、低星逆袭、废卡养成、普通宠物进化、破装备觉醒、失败角色翻盘、低资源开局逆袭等素材。

### 为什么有效

用户对“被低估的东西最后变最强”有天然代入感。关键不是让它立刻变强，而是让用户先看到它为什么值得被留下。

### 使用检查问题

- 主角是否是被迫接受坏局，而不是主动犯蠢？
- 被低估资产一开始是否真的看起来弱？
- 用户是否能理解它为什么被别人看不起？
- 主角是否有“不离不弃”的动作？
- 变强过程是否有阶段性反馈？
- 结尾是否证明它超过了原本更高价值的资产？

### 失效风险

- 开头铺垫太长，钩子不直接。
- 弱小资产太低质，损害产品高级感。
- 逆袭没有过程，只靠结尾突然变强。
- 困难只是折磨角色，没有转化为成长资源。

一句话总结：让被低估的资产在坏局里找到隐藏成长线，最后证明它才是真正的高价值选择。

## 方法 3：短反馈链法

### 核心结构

```text
困难 1 -> 小反馈 1
困难 2 -> 小反馈 2
困难 3 -> 小反馈 3
最终困难 / 对照 -> 大兑现反馈
```

### 适合用在什么上面

适合 20-40 秒剧情广告、养成进化、逃亡冒险、逆袭翻盘、闯关成长类素材。

### 为什么有效

短视频广告不能把爽点全部押到最后。如果中间只有铺垫，用户会在最终兑现前流失。每个困难后都要有可见变化，让用户持续获得“这条线有进展”的信号。

### 使用检查问题

- 中段是否持续提供可见反馈？
- 反馈是否能用画面理解，而不是靠文字解释？
- 关键动作是否都有反馈？
- 反馈之间是否递进，而不是重复？
- 小反馈是否服务最终大兑现反馈？
- 有没有一段只是在赶路、铺垫或等待？

### 失效风险

- 反馈太密，主线不清。
- 每个反馈都太弱，用户感受不到进展。
- 反馈之间没有因果，像随机事件拼贴。
- 中间反馈过强，抢走最终兑现反馈。

一句话总结：不要把爽点全部押到结尾；每个困难后都要给用户一个看得见的小回报。

## 方法 4：对照组闭环法

### 核心结构

```text
开头建立对照组
-> 对照组拥有更高资源 / 更强资产 / 更高地位
-> 主角处于低位
-> 主角经历成长
-> 结尾回到同一对照组
-> 用体型、战力、结果完成反转
```

### 适合用在什么上面

适合逆袭、打脸、竞技、成长、复仇、排名提升、公会战、阵容对抗等素材。

### 为什么有效

没有对照组，结尾只是“变强了”。有对照组，结尾才是“逆袭了”。

### 使用检查问题

- 开头是否明确建立谁强、谁弱？
- 对照组是否能在结尾再次出现？
- 结尾是否用画面证明主角超过了对照组？
- 反转是否回到了开头矛盾？
- 对手是否只是辅助主角，而没有抢走产品卖点？

### 失效风险

- 对手刻画太重，用户记住对手而不是产品。
- 结尾没有回到开头人物，逆袭感变弱。
- 只用数值对比，没有视觉证据。
- 打脸太生硬，降低角色代入。

一句话总结：开头谁压你，结尾就让谁亲眼看见你赢。

## 方法 5：夸张视觉刺激法

### 核心结构

```text
真实卖点
-> 极端放大
-> 形成体型、速度、数量、破坏力或稀有度冲击
-> 让用户产生继续观看和下载冲动
```

### 适合用在什么上面

适合巨大化、进化、秒杀、爆炸、战力暴涨、稀有角色登场、Boss 压迫、大规模战斗等素材。

### 为什么有效

用户往往不是先理解复杂设定，而是先被强刺激拦住。夸张表达能把“看起来还不错”变成“一眼就想继续看”。

### 使用检查问题

- 画面是否足够大、爽、危险、离谱？
- 用户是否能在 1 秒内感受到差距？
- 夸张是否服务产品卖点？
- 是否有明确对比对象来放大刺激？

### 失效风险

- 夸张过头，和产品真实体验断裂。
- 只有视觉刺激，没有玩法或转化动机。
- 全程都很炸，最终兑现反馈反而不突出。

一句话总结：把卖点视觉化、极端化，让用户先被画面抓住。

## 方法 6：反常识结果互动法

### 核心结构

```text
看起来正确的行为 -> 得到普通 / 平庸结果
看起来错误的行为 -> 得到隐藏 / 高价值结果
```

### 适合用在什么上面

适合选择题、孵化、合成、开箱、挑战、解谜、隐藏配方、特殊条件和禁忌选择类素材。

### 为什么有效

反常识结果会制造停留、评论和代入。用户会想判断“为什么会这样”，也会产生“如果是我，我会怎么选”的互动感。

### 使用检查问题

- 正确行为得到的结果是否不是失败，而是“正确但平庸”？
- 错误行为得到高价值结果是否有隐藏解释？
- 主角是否避免显得主动犯蠢？
- 反常识是否能和被迫坏局结合，让反转更自然？

### 失效风险

- 反转缺乏解释，显得硬拗。
- 主角主动犯蠢，降低代入。
- 误导真实玩法，引发落差。

一句话总结：让用户以为自己知道答案，再用隐藏规则推翻他。

## 方法 7：挑战失败诱导法

### 核心结构

```text
看似简单挑战
-> 操作者连续犯错或差一点成功
-> 结尾失败
-> 用户觉得“我上肯定能过”
```

### 适合用在什么上面

适合闯关、解谜、跑酷、塔防、操作挑战、差一点通关的关卡素材。

### 为什么有效

挑战类素材里，成功结尾不一定最强。失败结尾能激发用户的纠错欲、胜负心和操作自信，从而促使下载尝试。

### 使用检查问题

- 挑战目标是否在开头可理解？
- 失败是否让用户觉得操作者太菜，而不是游戏太难？
- 用户是否能立刻想到自己会怎么做？
- 结尾失败是否留出重新尝试的冲动？

### 失效风险

- 失败原因不清楚，用户只会困惑。
- 操作过难，用户觉得自己也过不了。
- 失败太惨或太蠢，降低产品质感。

一句话总结：挑战类视频可以故意失败，让用户带着“我能赢”的心理下载。

## 方法 8：故事冲突驱动法

### 核心结构

```text
明确目标
-> 强阻碍 / 对手 / 压力 / 损失风险
-> 主角采取行动
-> 中途出现假失败或更大危机
-> 通过核心卖点解决冲突
-> 给出情绪兑现和下载动机
```

### 适合用在什么上面

适合脚本逻辑正确但观感平、玩法承接变成功能演示、收集养成和战斗系统缺情绪动机的素材。

### 为什么有效

冲突会让用户替主角着急、替资产不服、替结果期待。下载冲动不是来自“我知道这个功能”，而是来自“我也想进去解决这个问题”。

### 常见冲突来源

- 对手压力：强玩家、Boss、敌方阵营、排行榜第一。
- 时间压力：倒计时、最后一次机会、城门将破。
- 资源压力：只剩一个道具、只差最后一格、战力不足。
- 身份压力：被嘲笑、被赶走、被低估、队友失望。
- 结果压力：失败就会错过奖励、输掉团战、无法补齐阵容。

### 使用检查问题

- 主角当前想要什么？
- 什么东西阻止他得到？
- 如果失败，会损失什么？
- 冲突是否在前几秒被看懂？
- 结尾是否用产品核心卖点解决冲突？
- 用户看完是否会产生“我也想试”的动作欲望？

### 失效风险

- 冲突太弱，没有压力和损失。
- 冲突太复杂，开头看不懂。
- 冲突和产品脱节。
- 只有负面压力，没有足够兑现反馈。

一句话总结：脚本必须有故事冲突；冲突推动故事继续，情绪推动用户下载。

## 方法 9：素材即定位法

### 核心结构

```text
目标高质量用户是谁
-> 他为什么愿意长期玩 / 付费
-> 素材放大这些动机信号
-> 广告系统学习相似用户
-> 用后端数据校验用户质量
```

### 适合用在什么上面

适合广告平台定向弱化、CTR 还可以但后端质量不理想、同题材下需要区分泛用户和核心用户的素材策略。

### 为什么有效

素材不是“把功能讲清楚”而已，而是在告诉广告系统：请帮我找会被这些幻想、目标、压力和奖励吸引的人。

### 常见定位信号

- 题材信号：魔法、战争、末日、现代、古代、科幻。
- 收集信号：图鉴、未解锁槽位、稀有度、补齐奖励。
- 养成信号：升级、升星、技能解锁、战力成长。
- 竞争信号：排行榜、公会战、Boss、攻城、反超。
- 炫耀信号：稀有资产展示、别人震惊、主城展示。
- 付费倾向信号：长期目标、资源差距、社交竞争、战力追赶。

### 使用检查问题

- 这条素材想吸引哪类用户？
- 哪些画面在筛选这些用户？
- 素材有没有吸引大量低质量泛用户的风险？
- 如果 CTR 高但 ROI 弱，是不是概念幻想过强、付费动机过弱？
- 如果 CTR 低但 ROI 好，是否应保留为窄受众高质量素材？

### 失效风险

- 只追求高 CTR，素材过泛。
- 只做重度玩法表达，开头理解成本过高。
- 素材信号互相冲突。
- 素材承诺与真实体验断裂。

一句话总结：素材不是单纯展示产品，而是在替算法选择用户。

## 方法 10：触发机制可信化法

### 核心结构

```text
关键变化
-> 外部刺激 / 隐藏规则 / 能量来源 / 资源消耗
-> 变化发生
-> 变化结果进入产品核心收益
```

### 适合用在什么上面

适合弱小单位变强、普通道具变稀有、失败局面逆袭、隐藏职业觉醒、合成升级、进化突破等素材。

### 为什么有效

效果广告可以夸张，但关键变化不能像“剧情需要”。用户不需要理解复杂设定，但需要看懂一个简单因果：它不是凭空变强，而是因为某个刺激触发了隐藏能力。

### 常见触发机制

- 外部刺激：敌方攻击、雷电、火焰、Boss 技能、环境压力。
- 隐藏规则：血脉识别、图鉴共鸣、禁忌配方、特殊环境条件。
- 能量来源：灵魂、结晶、已收集单位、Boss 掉落、祭坛能量。
- 资源消耗：喂养、升级、升星、合成、投入稀有材料。
- 情绪触发：保护主人、濒死、守城、最后一战。

### 使用检查问题

- 核心变化是什么？
- 变化前后差异是否足够强？
- 触发它的外部刺激是什么？
- 能量或资源从哪里来？
- 这个变化为什么和产品核心功能有关？
- 用户是否能在 1-2 个画面内看懂因果？

### 失效风险

- 解释太复杂，用户看不懂。
- 触发机制和产品无关。
- 外部刺激太弱，不足以支撑巨大变化。
- 只有特效，没有明确来源。

一句话总结：关键变化必须有触发机制；用户看懂它为什么发生，才会相信这个爽点。

## 复盘用十问

```text
1. 用户为什么想继续看下一个结果？
2. 用户为什么会替主角或资产站队？
3. 中间每几秒给一次可见反馈？
4. 结尾是否回到了开头矛盾并完成反转？
5. 画面有没有足够夸张的视觉刺激？
6. 有没有能制造互动的反常识结果？
7. 挑战类素材是否激发“我能过”的心理？
8. 故事冲突是否足以调动情绪并制造下载冲动？
9. 这条素材在吸引谁、过滤谁？
10. 核心变化是否有清晰触发机制？
```

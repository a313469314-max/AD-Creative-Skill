# 广告创意方法调用流程（对外分享版）

## 适合用在什么上面

适合用于游戏广告创意、脚本方向、分镜、BGM 建议、素材复盘和参考视频迁移任务。

这个流程的作用不是替代创意判断，而是规定在不同任务里，应该按什么顺序调用方法论，避免每次创意都随机拼方法。

## 核心原则

方法论是分层工具箱，不是固定模板。每次任务先明确：

```text
产品真实承接是什么？
素材目标是什么？
当前问题是什么？
需要调用哪类方法？
哪些方法不适合本次任务？
```

默认优先级：

```text
产品真实承接
> 广告数据和已验证素材
> 用户调研 / 问卷 / 访谈
> 方法论
> 创意发散
```

方法论和产品真实承接冲突时，以产品真实承接为准。方法论和已验证素材数据冲突时，先标注冲突点，再由业务负责人判断是否临时覆盖。

## 钩子、承接和产品证明

“产品真实承接优先”不等于“开头钩子必须贴产品”。

开头钩子、承接桥和产品证明要按阶段任务判断，而不是按固定秒数切段：

- 钩子段：第一任务是吸引注意力，让用户停下来。
- 承接桥：把钩子的注意力转译到产品资产、产品幻想或玩法动作。
- 产品证明：证明下载后有对应体验，不是纯噱头。

强钩子可以借用外部场景、强情绪、强感官、异常动作或平台原生内容形态，但跨世界观、跨题材、跨风格时，必须有明确桥接机制，例如梦境、直播试玩、屏幕进入游戏、实验召唤、现实道具通向游戏世界等。

不能无解释硬切。

## 开头切入原则

效果广告开头优先从小事件切入，而不是宏大场景叙事。

优先使用：

- 小异常。
- 小失败。
- 小选择。
- 近景道具。
- 个人处境。
- 即时倒计时。
- 用户一眼能懂的危机。

复杂系统、多人组织、大规模战场、世界观背景、长期目标和重度玩法规则，应放在中后段，用作压力放大、产品证明或高质量用户筛选信号。

## 默认调用顺序

### 步骤 1：读取产品上下文

先确认产品真实承接、目标用户、核心卖点、美术资产和指标口径。

建议至少读取或确认：

- 产品品类和真实玩法。
- 核心循环和可展示功能。
- 目标市场和目标用户。
- 当前广告目标：CTR、CPI、CVR、ROI、留存、付费率或素材扩量。
- 已验证素材和历史数据。
- 美术资产、角色资产、场景资产和禁用边界。
- 平台审核、品牌安全和制作成本限制。

用户问卷或访谈只能作为样本参考，不能直接外推为全量买量用户画像。

### 步骤 2：判断任务类型

按任务类型选择方法，不默认全读全套。

| 任务 | 默认调用 |
| --- | --- |
| 创意方向 / 脚本生成 | 效果广告创意总纲 + 方法卡片库 + 内容创作迁移 + 优秀案例分析 |
| 参考视频拆解 / 迁移 | 参考视频两阶段流程 + 素材拆解工作流 + 优秀案例分析 + 效果广告创意总纲 |
| 分镜图 / 出图 prompt | 游戏概念分镜提示词方法 + 产品美术资产约束 |
| BGM / 镜头节奏 | 优秀案例分析中的 BGM、节奏画面和音画同步方法 |
| 素材复盘 / 数据判断 | 产品指标口径 + 素材即定位 + 已验证素材 |
| 新案例库分析 | 优秀案例分析中的新增案例分析模板 |

### 步骤 3：用总纲做基础诊断

先用“效果广告创意方法论总纲”检查：

- 开头钩子是否能抓停留。
- 承接桥是否能把钩子转回产品幻想。
- 脚本是否有故事冲突。
- 素材是否承担用户定位。
- 核心变化是否有触发机制。
- 关键动作是否有可见反馈。
- 中段反馈是否递进。
- 单镜头容量是否可制作。

### 步骤 4：按问题调用方法卡片

用“效果广告创意方法卡片库”按问题选择具体方法：

| 当前问题 | 优先方法 |
| --- | --- |
| 缺好奇心 / 收集欲 | 同源多结果 |
| 缺情绪代入 / 弱变强 | 被低估资产逆袭 |
| 中段拖 | 短反馈链 |
| 结尾不爽 | 对照组闭环 |
| 画面不刺激 | 夸张视觉刺激 |
| 选择感和评论欲不足 | 反常识结果互动 |
| 挑战类素材缺下载冲动 | 挑战失败诱导 |
| 像功能说明书 | 故事冲突驱动 |
| CTR 好但 ROI / 留存 / 付费差 | 素材即定位 |
| 关键转折牵强 | 触发机制可信化 |

### 步骤 5：用内容创作方法找角度和情绪

用“内容创作方法迁移广告创意”检查：

- 是否做到情理之中、预料之外。
- 功能是否被转成故事动作。
- 用户情绪把柄是什么。
- 情绪地图是否清楚。
- 是否有三番递进。
- 是否只是信息搬运，而不是创意表达。

这个步骤主要解决“同一个卖点从什么角度切入”的问题。

### 步骤 6：用优秀案例方法选择表达机制

用“优秀素材案例分析方法论”检查：

- 是否适合 BGM 驱动。
- 是否适合包装成直播、发布会、道歉流、采访、战报、梗图等内容形态。
- 是否需要强刺激钩子，以及风险边界在哪里。
- 镜头节奏是否和音乐能量匹配。
- 是否可以借用验证过的脚本结构。
- 学的是底层机制，还是只是在照搬表层题材。

优秀案例库是创意机制库，不是题材照搬库。

### 步骤 7：进入分镜和 prompt 执行

如果任务进入分镜图或出图 prompt，再调用“游戏概念分镜提示词方法”。

分镜和 prompt 必须按真实游戏体验链路组织画面，写清：

- 画面用途。
- 画面规格。
- 镜头语言。
- 场景设定。
- 主体角色。
- 构图关系。
- 玩法 UI。
- 美术风格。
- 色彩体系。
- 质量约束。

分镜不是氛围图堆叠。每一帧都要像游戏流程中的一个阶段，有玩家目标、玩法状态、反馈和前后承接。

## 参考视频任务的特殊规则

所有“根据参考视频构思创意”的任务，默认先进入故事方向筛选，不直接进入逐镜头脚本、时间码脚本、出图 prompt 或制作目录。

标准顺序：

```text
原视频场景变化分镜
-> 原视频底层结构和可迁移点
-> 候选故事方向筛选池
-> 人类认可某个方向
-> 分镜级脚本变体
-> prompt / 出图 / 版本迭代
```

第一阶段只输出候选故事方向，禁止提前输出逐镜头分镜、出图 prompt、关键帧画面描述或过细时间码脚本。

第一阶段每个方向至少回答：

- 核心假设是什么。
- 故事钩子是什么。
- 冲突如何升级。
- 关键变化由什么触发。
- 如何承接产品核心功能。
- 适合验证什么指标。
- 主要风险是什么。
- 人类需要判断什么。

只有当人类明确采用某个方向后，才进入第二阶段分镜级脚本。

## 最小输出要求

每次正式产出创意时，应简短说明：

```text
本次调用了哪些产品上下文？
本次主要诊断问题是什么？
本次采用了哪些方法？
哪些方法被排除，为什么？
```

每个正式创意方案输出前，至少回答：

```text
这个方案吸引谁？
过滤谁？
钩子段靠什么停留？
钩子是否足够强到值得保留？
承接桥如何把钩子转译到产品幻想？
核心变化为什么可信？
哪个指标验证钩子？
哪个指标验证用户质量？
哪个指标验证可加码？
```

## 禁止用法

- 不读产品上下文，直接套方法。
- 因为某个案例爆，就照搬表层题材。
- 把开头钩子当成产品卖点优先级。
- 用热点、梗图或强刺激替代产品核心。
- 只写“优化节奏”“增强情绪”，不说明具体镜头、指标和判断标准。
- 只追求 CTR，不检查 CPI、CVR、ROI、付费率、留存和 LTV 承接。
- 分镜 prompt 只写氛围，不写玩法状态和 UI 反馈。

## 一句话总结

这个流程的核心是：

```text
先确认产品真实承接
再判断素材目标和当前问题
再调用对应方法
最后用指标验证创意是否真的成立
```
